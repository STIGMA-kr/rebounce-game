# Rebound Node

전통적인 벽돌깨기를 뒤집은 웹 게임 프로토타입입니다. 적이 먼저 공을 발사하고, 플레이어는 6개 고정 위치 중 하나로 노드를 순간이동시켜 공을 튕겨냅니다.

## 실행

1. VS Code에서 이 폴더를 엽니다.
2. `index.html`을 브라우저로 열면 바로 실행됩니다.
3. Live Server 확장을 쓰는 경우 `index.html`에서 `Open with Live Server`를 선택해도 됩니다.

빌드 과정이나 패키지 설치는 필요 없습니다.

## 조작

- `A S D L ; '`: 왼쪽부터 오른쪽까지 6개 위치로 노드 이동
- `Enter` 또는 `Space`: 시작, 일시정지, 재개
- 화면 상단 `▶`, `Ⅱ` 버튼도 사용할 수 있습니다.
- 노드는 원형으로 표시되지만, 실제 반사 판정은 선택된 위치의 하단 직사각형 구역 전체입니다.

## 게임 규칙 반영 상태

- 플레이어 HP는 10으로 시작합니다.
- 잡몹 라운드는 6x6 그리드에 적이 배치됩니다.
- 잡몹 라운드에서는 발사 순간이 겹치지 않지만, 화면 안에는 `현재 레벨 + 2`개까지 공이 동시에 존재할 수 있습니다.
- 잡몹의 다음 발사 간격은 매번 랜덤으로 정해집니다.
- 잡몹 공은 해당 열의 노드 위치로 날아오며, 선택된 노드 구역에 닿으면 위쪽 잡몹 그리드로 되돌아갑니다.
- 반사된 잡몹 공은 발사자뿐 아니라 경로상 닿은 어떤 살아있는 잡몹에게도 피해를 줄 수 있습니다.
- 반사된 잡몹 공이 잡몹을 맞추지 못하면 위쪽과 좌우 벽에서 튕기며 유지됩니다.
- 잡몹 공이 하단 플레이어 데미지 구역에 닿기 전 선택된 노드 구역으로 막지 못하면 공이 사라지고 HP가 감소합니다.
- 모든 잡몹을 제거하면 보스 라운드로 넘어갑니다.
- 보스는 노드가 현재 있지 않은 위치를 향해 공을 발사합니다.
- 보스 라운드에서 동시에 존재할 수 있는 공의 최대 개수는 `현재 레벨 + 1`개입니다.
- 보스 공은 원형, 막대기, 삼각형, 사각형 중 하나의 모양으로 등장합니다.
- 막대기, 삼각형, 사각형 보스 공은 벽, 노드, 보스에 튕길 때 회전합니다.
- 보스 라운드 공은 선택된 노드 구역에 닿으면 반드시 튕기고, 보스에게 맞아도 사라지지 않고 계속 튕깁니다.
- 보스 공은 튕길수록 빨라지며, 레벨에 따른 속도 상한이 있습니다.
- 잡몹과 보스는 공을 발사하는 순간 발사 이미지로 짧게 바뀌고, 발사 방향의 반대로 밀리는 반동을 보여줍니다.
- 보스를 처치하면 다음 레벨로 넘어가고 구조가 무한 반복됩니다.

## 난이도 조절

`js/game.js` 상단 근처의 함수들을 수정하면 됩니다.

- `mobHealthForLevel()`: 잡몹 체력
- `mobCountForLevel()`: 잡몹 수
- `mobFireInterval()`: 잡몹 발사 간격의 기준값
- `mobFireDelay()`: 잡몹 발사 간격 랜덤 범위
- `mobBallLimit()`: 잡몹 라운드 최대 공 개수
- `mobBallsInPlay()`: 잡몹 라운드에서 현재 존재하는 공 개수 계산
- `mobShotSpeed()`: 잡몹 공 이동 속도
- `bossHealthForLevel()`: 보스 체력
- `bossFireInterval()`: 보스 발사 간격
- `bossBallLimit()`: 보스 라운드 최대 공 개수, 기본값은 `현재 레벨 + 1`
- `bossSpeedCap()`: 보스 공 속도 상한
